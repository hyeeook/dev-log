- [x] My K 파트에서 KOS-OL을 통해 안면인증 API 호출 테스트

---

### ⚠️ 문제점 & 해결
- 현황<br>
    > KOS-OL을 통한 기준정보조회API 호출 테스트 중 PostMan을 통해 호출하면 정상 호출된다.
- 문제<br>
    > PostMan과 동일한 Input으로 개발 tool에 데이터를 하드코딩하여 호출 테스트를 하면 Output이 없는 현상이 발생했다.<br>
    > 제공한 API는 총 세 종류이며, 그 중 기준정보조회API만 해당 문제점 발생했다.
- 원인<br>
    > Input 데이터 중 trtDivCd 필드가 있는데, 개발 tool에서 테스트할 때는 잘못된 필드명(triDivCd)을 사용했다.
- 해결<br>
    > 1차 : 개발 tool로 호출할 때 header 또는 body 구성을 잘못한 것은 아닌지 My K 소스 확인했으나 문제가 될 내용은 없었다.<br>
    > 2차 : response body의 명칭이 잘못된 것은 아닌지 확인했으나 올바른 명칭을 사용했었다.<br>
    > 3차 : B-MON 사이트에서 개발 tool로 호출할 때의 Input 데이터를 가져와서 PostMan으로 호출 테스트 진행했으며, 그 결과 동일한 이슈 발생하여 Input 필드명이 잘못된 것을 확인하여 해결했다(KOS-OL 개발자 통해 확인).

---

### 💡 새로 배운 것
- 이슈가 발생했을 때 반드시 시간이 걸려도 원인을 파악해야 한다.
    > B-MON 로그를 통해 Output이 null인 것을 확인했으나, 구체적으로 Output이 null인 것이 무엇이 잘못되었고 null 값 때문에 어떤 현상이 발생한 것인지 생각하지 못했다.<br>
    > 조회 결과가 없으니 Output 필드들이이 null로 나온 것이고, Output 필드들이 null이므로 KOS-OL 호출 테스트 결과 response body가 빈 값으로 나오는 것이었음을 먼저 파악을 했어야 했다.
